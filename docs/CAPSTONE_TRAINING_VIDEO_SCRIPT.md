# CapStone Training Video Script

This script is designed for recording a practical CapStone walkthrough. It can be used as narration, a shot list, and a checklist for the person recording the video.

Estimated video length: 10 to 15 minutes.

## Training goals

By the end of the video, the viewer should know how to:

- open CapStone
- refresh and select a Zoho deal
- capture GPS
- record or enter field notes
- add photos and descriptions
- use the optional report fields
- generate an AI report
- save/update the Zoho deal note
- understand WorkDrive uploads
- reopen a report from History

## Before recording

Prepare:

- a test Zoho deal
- one or two sample photos
- permission to use GPS/location
- a short sample field note
- a browser opened to the latest CapStone version URL

Example URL:

```text
https://BJWCAC.github.io/fieldpro/FieldPro.html?v=127
```

Replace `127` with the latest build number when recording.

Use a test deal if possible so the training does not create real customer records.

## Video outline

### 1. Introduction

Screen action:

- Show the CapStone home screen.

Narration:

> This is CapStone by Calibrations & Controls. CapStone is used to document field work, generate a professional AI field service report, save the report to Zoho deal notes, and upload supporting files to WorkDrive.

Key points:

- CapStone connects the field report to a Zoho deal.
- It uses notes, photos, GPS, and optional structured fields.
- Reports can be reopened from History and updated later.

### 2. Refresh and select a deal

Screen action:

- Go to the Deals tab.
- Tap Refresh from Zoho.
- Search/filter if needed.
- Select a test deal.

Narration:

> Start by refreshing deals from Zoho. After the deals load, select the deal for the site visit. The active deal is important because it controls where the report note is saved in Zoho and how the WorkDrive files are organized.

Callout:

- Confirm the active deal appears in the header and report context area.

### 3. Capture GPS location

Screen action:

- Go to Capture.
- Tap Get Location.
- Allow location permission if prompted.

Narration:

> Next, capture the site location. CapStone stores the GPS coordinates and, when available, the address. This information is included in the report header and Zoho note.

Callout:

- Show GPS coordinates appearing in the location/report context fields.

### 4. Add field notes

Screen action:

- Tap the Voice Notes field.
- Dictate or type a sample note.

Sample note:

```text
Performed site visit at the lift station. Checked control panel operation, verified pump lead-lag sequence, inspected float wiring, and confirmed alarm light operation. Found corrosion on one terminal strip and recommend replacement during next service visit.
```

Narration:

> Add the main field notes here. You can type or use dictation. The AI report is based heavily on what you describe, so include the important facts, work performed, findings, and recommendations.

Tip:

- Be specific.
- Do not rely on the AI to invent missing details.

### 5. Add photos

Screen action:

- Add one or more photos.
- Add a short description under each photo.

Narration:

> Photos can be captured or uploaded from the gallery. Add a short description to each photo so CapStone knows why the photo matters.

Example photo descriptions:

- Control panel overview after inspection.
- Corrosion observed on terminal strip.
- Pump controller status screen showing normal operation.

### 6. Explain the nine optional report fields

Screen action:

- Scroll to the nine report fields.
- Fill one or two fields as an example.

Narration:

> These nine fields are optional. You do not need to fill them all out. If they are blank, CapStone will build the report from your notes, photos, GPS, and deal information. Use these fields when you want to force information into a specific report section.

Example:

- Issues / Deficiencies: `Corrosion noted on terminal strip.`
- Recommendations & Next Steps: `Replace terminal strip during next scheduled service.`

Callout:

- Empty fields are ignored.
- Filled fields guide the AI report structure.

### 7. Generate the AI report

Screen action:

- Tap Generate AI Report.
- Wait for report generation.
- Show the report screen.

Narration:

> When the capture information is ready, tap Generate AI Report. CapStone sends the information to the AI service and builds a professional field service report.

Callout:

- Review the report before relying on it.
- Regenerate if more details are added or corrections are needed.

### 8. Review the generated report

Screen action:

- Scroll through Report Header, photos, and report body.

Narration:

> Review the report header, site information, photos, and the report text. Confirm the account, deal, GPS, findings, and recommendations are correct.

Checklist:

- Correct account
- Correct deal
- Correct GPS/site
- Work performed is accurate
- Findings are accurate
- Recommendations are accurate

### 9. Save to Zoho

Screen action:

- Tap Save to Zoho if it did not auto-save.
- Show save status.

Narration:

> Saving to Zoho creates or updates the deal note. CapStone is designed to avoid duplicate notes for the same report. The first save creates the note. If you regenerate the same report later, CapStone updates that existing note.

Important explanation:

> If the note was deleted directly in Zoho, CapStone will create a replacement note the next time the report is saved.

### 10. Explain WorkDrive files

Screen action:

- Show upload status if visible.
- Optionally open WorkDrive outside the app if appropriate.

Narration:

> CapStone also uploads report files, photos, and video to WorkDrive. For the same report, file names are stable so regenerating the report should replace matching files instead of creating repeated timestamped copies.

Callout:

- Existing old duplicates may need manual cleanup.
- Future regenerations should be cleaner.

### 11. Download PDF

Screen action:

- In PDF Options, show Include Photos toggle.
- Tap Download PDF.

Narration:

> You can download a PDF copy of the report. Use the Include Photos option to control whether photos are included in the PDF.

### 12. Continue a report from History

Screen action:

- Go to History.
- Open a previous report.
- Tap Open + Continue.
- Add or change a note.
- Regenerate.

Narration:

> Reports are saved in local History. If you need to continue a report, open it from History, make changes, and regenerate. CapStone will attempt to update the existing Zoho note for that report instead of creating a duplicate.

Important note:

> History is stored in the browser. Clearing browser data can remove local History.

### 13. Troubleshooting section

Screen action:

- Show Settings.
- Point out reset/cache options.

Narration:

> If the app does not look updated, open the latest versioned URL. If deals seem stale, refresh from Zoho or clear cached deals. If WorkDrive upload is slow, the Zoho note save should still continue.

Troubleshooting examples:

- App did not update: use the latest `?v=` URL.
- Deals do not load: retry Refresh from Zoho.
- Duplicate old WorkDrive files: manually clean old duplicates.
- Zoho note deleted manually: save again from CapStone to create a replacement.

### 14. Closing

Screen action:

- Return to the report screen or CapStone logo.

Narration:

> That is the standard CapStone workflow: select the deal, capture location and notes, add photos, generate the report, review it, and save it back to Zoho and WorkDrive.

## Quick-start checklist for technicians

1. Open CapStone.
2. Refresh Zoho deals.
3. Select the correct deal.
4. Capture GPS.
5. Add voice notes or typed notes.
6. Add photos and descriptions.
7. Fill optional report fields only when needed.
8. Generate the AI report.
9. Review the report.
10. Save/update Zoho.
11. Download PDF if needed.
12. Use History to continue the report later.

## Short version script

Use this if recording a shorter 3 to 5 minute video.

> CapStone helps document field service work and save reports to Zoho and WorkDrive. Start by refreshing Zoho deals and selecting the correct deal. Capture GPS, then add notes, photos, and optional section details. The nine report fields are optional; use them only when you want specific information in a specific section. Tap Generate AI Report, review the output, and save it to Zoho. If you regenerate the same report later, CapStone updates the existing Zoho note and reuses stable WorkDrive filenames to reduce duplicates. Reports can be reopened from History and continued later.

## Recommended recording notes

- Record in a quiet place.
- Use a test deal.
- Avoid showing private customer information.
- Keep the walkthrough slow enough that each button click is visible.
- Mention that CapStone should be reviewed by the technician before final use.
